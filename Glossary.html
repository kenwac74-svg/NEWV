LONGRISE 메뉴 용어집

용어 통일은 추후 일괄 진행됩니다. 그 전까지 개발자가 혼선을 겪지 않도록 구 용어 → 신 용어 매핑을 정리합니다. 내부 라우트 키·컴포넌트 이름은 변경하지 않아 코드 영향이 최소화됩니다 — 표시 라벨만 바뀝니다.

PC ↔ 모바일은 1:1 매칭이 아닙니다. 동일 개념이라도 라벨이 다릅니다 — PC Packages = 모바일 Plans = 내부 PLANS, PC Rewards = 모바일 Network = 내부 NETWORK, PC Dashboard = 모바일 Earn = 내부 EARN. 개발 시에는 라벨이 아닌 내부 키를 기준으로 분기하세요.

2. 모바일 하단 네비게이션 순서

PLANS → NETWORK → EARN → WALLET → ALL(전체) · 기본 착지 = EARN. 사이드바/드로어는 사용하지 않고 ALL 탭이 전체 핫링크 허브를 대체합니다. PC에서는 ALL 대신 Sitemap 페이지 (푸터 + 로그인 아이콘 서브메뉴의 로그아웃 바로 위)로 제공합니다.

3. 로그인 직후 오버레이 순서

  • ① 동의 게이트 (매 로그인) — Risk Notice + Terms 2단 박스, 각각 개별 동의. 내용이 길면 스크롤(전부 내릴 필요 없음), 장식 없는 텍스트. 둘 다 체크해야 진행.
  • ② 중요 공지 팝업 (선택) — LR_DATA.importantNotice가 있을 때만 1개 더 표시. 없으면 생략.
  • ③ Start Here 유도 (신규/미구매) — 구매 이력 없는 사용자에게만 표시, Start Here 페이지로 유도.

데모 토글 — Tweaks 패널에서 User: new / active, Important notice popup on/off, Replay login flow로 시퀀스를 재현할 수 있습니다. 실제 구현에서는 user.hasPurchased 등 계정 상태로 ③ 노출을 분기하세요.

4. EARN(대시보드) 액션 정리

  • EARN 페이지에서 Deposit / Withdraw / Send / Swap 제거 — Wallet과 중복이므로 Wallet으로 일원화.
  • 대신 보유 패키지 목록(패키지별 수익·수익률·진행률)을 표시.
  • 단일 Get more Profit 버튼 → PLANS 이동.

5. START HERE 처리

  • 고정 메뉴가 아님 — 모바일·PC 모두 홈(EARN) 상단 배너로 노출, START HERE.html로 링크.
  • 모바일 신규/미구매 사용자: 로그인 노티 팝업 이후 ③ Start Here 유도 팝업으로 진입.
